## Extended material

- `plans/` — dated planning and design artifacts
- `plugins/` — plugin specs and authoring references
- `spec/` — subsystem-specific technical specs
- `experimental/` — exploratory technical notes
- `assets/` — documentation assets

`plans/` and most `experimental/` docs are archival/reference material and are intentionally not part of top-level docs navigation.

## Relationship to `docs/` guides

- `docs/deploy/*` provides concise operator-focused instructions.
- `docs/reference/*` provides deeper implementation and maintainer context.

When both exist for a topic, treat `docs/deploy/*` as the quick path and `docs/reference/*` as the deep path.
